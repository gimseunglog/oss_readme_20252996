### 1. top

+ **top**: `top` 명령어는 리눅스 시스템의 **운용 상태를 실시간으로 모니터링**하고 프로세스를 관리할 수 있는 강력한 유틸리티이다. 윈도우의 작업 관리자와 유사하게, **CPU 사용량, 메모리 사용량, 실행 중인 프로세스 목록** 등 시스템 전반의 부하 상황을 주기적으로 업데이트하여 보여준다.

+ 명령어 실행
```bash
top
```

+ 실행 화면
<img width="2537" height="568" alt="스크린샷 2025-11-19 161443" src="https://github.com/user-attachments/assets/6f7eb448-30b5-4ffc-80b1-de78cbfebefe" />

+ 각 열의 의미
  
| 컬럼      | 설명                        |
| ------- | ------------------------- |
| PID     | 프로세스 ID                   |
| USER    | 실행 사용자                    |
| PR      | 우선순위                      |
| NI      | nice 값 (우선순위 조정 가능)       |
| VIRT    | 가상 메모리 사용량                |
| RES     | 실제 메모리 사용량                |
| SHR     | 공유 메모리                    |
| S       | 상태 (S=sleep, R=running 등) |
| %CPU    | CPU 사용량                   |
| %MEM    | 메모리 사용량                   |
| TIME+   | 사용한 CPU 시간                |
| COMMAND | 명령어 이름                    |

+ `top` 단축키

| 단축키   | 기능                  |
| ----- | ------------------- |
| **q** | top 종료              |
| **h** | help                |
| **1** | CPU 코어별 사용량 표시      |
| **P** | CPU 사용률 기준 정렬       |
| **M** | 메모리 사용률 기준 정렬       |
| **T** | 사용 시간 기준 정렬         |
| **k** | 프로세스 종료 (PID 입력 필요) |
| **u** | 특정 사용자 프로세스만 보기     |
| **z** | 컬러 모드 토글            |
| **d** | refresh 간격 변경       |

+ `top` 옵션 정리

| 옵션 | 설명 |
|------|-------|
| `-p <PID>` | 특정 PID만 모니터링 |
| `-u <USER>` | 특정 사용자 프로세스만 표시 |
| `-d <초>` | 갱신(refresh) 간격 설정 (기본 3초) |
| `-b` | 배치 모드 (로그 저장용, 화면 UI 없음) |
| `-n <횟수>` | 출력 횟수 지정 (`-b`와 함께 자주 사용) |
| `-o <정렬기준>` | 특정 컬럼 기준 정렬해서 시작 (예: `%CPU`, `%MEM`) |
| `-i` | idle 프로세스(0% CPU) 숨기기 |
| `-H` | 스레드 단위(Thread mode)로 표시 |
| `-s` | 보안 모드(프로세스 정보 제한) |


### 2. ps

+ **ps**: `ps`는  현재 시스템에서 **실행 중인 프로세스를 확인**할 때 사용하는 명령어이다. 기본적으로 **프로세스의 상태, PID(Process ID), CPU 사용량** 등의 정보를 출력한다.

+ 명령어 실행
```bash
ps
```

+ 실행 화면
<img width="570" height="134" alt="제목 없음2" src="https://github.com/user-attachments/assets/2de21c64-88dd-4b6b-9f87-2e4bbe9abd0f" />

+ 각 열의 의미

| 열(Column) | 의미            |
| --------- | ------------- |
| **PID**   | 프로세스 ID       |
| **TTY**   | 프로세스를 실행한 터미널 |
| **TIME**  | CPU 사용 시간 합계  |
| **CMD**   | 실행 명령어        |


+ `ps` 옵션 정리

| 옵션                 | 설명                     |
| ------------------- | ---------------------- |
| `-e` / `-A`   | 시스템 전체 프로세스 보기         |            |
| `-f`             | 전체 정보(Full Format) 보기  |            |
| `-ef`            | 모든 프로세스를 전체 정보와 함께 보기  |            |
| `aux`            | BSD 스타일 전체 프로세스 보기     |            |
| `-u <user>`      | 특정 사용자 프로세스 보기         |            |
| `-eo <fields>`   | 출력 필드 지정               |            |
| `--sort=<field>` | 특정 필드로 정렬              |            |

### 3. jobs
+ **jobs**: `jobs`는 **현재 터미널에서 실행 중인 작업(백그라운드나 중지 상태로 실행된 프로세스)들의 상태**를 확인하는 명령어이다.

+ 명령어 실행
```bash
jobs
```

+ 실행 화면
+ 각 열의 의미

+ `jobs` 옵션 정리

| 옵션   | 설명                   |
| ---- | -------------------- |
| `-l` | PID(Process ID)까지 표시 |
| `-p` | 작업의 PID만 출력          |
| `-n` | 최근 상태가 변경된 작업만 출력    |
| `-r` | 실행 중인 작업만 표시         |
| `-s` | 중지된 작업만 표시           |

### 4. kill
+ **kill**: `kill`은 **프로세스를 종료하거나 시그널을 보내는** 명령어이다. 단순히 *죽이는* 용도뿐만 아니라, 다양한 시그널을 보내서 프로세스를 제어할 수도 있다.
+ 명령어 실행
```bash
bash
```
+ 실행 화면
+ 자주 쓰는 시그널

| 시그널       | 번호 | 설명                           |
| --------- | -- | ---------------------------- |
| `-SIGTERM` | `-15` | 정상 종료 요청 (프로세스가 정리 후 종료 가능)  |
| `-SIGKILL` | `-9`  | 강제 종료 (프로세스가 무조건 종료됨, 정리 불가) |
| `-SIGSTOP` | `-19` | 프로세스 일시 중지                   |
| `-SIGCONT` | `-18` | 일시 중지된 프로세스 재개               |
| `-SIGHUP`  | `-1`  | 터미널 종료 신호, 프로세스 재시작 용도로 사용   |
